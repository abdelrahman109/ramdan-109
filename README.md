# Telegram Event Bot System (بدون Paymob)

سيستم حجز تذاكر للحفلات عبر تيليجرام، مع QR Code صالح مرة واحدة، ودفع يدوي عبر InstaPay أو المحافظ الإلكترونية.

## الجديد في النسخة دي
- طلب **اسم الشخص القائم بالحجز** بشكل مستقل عن اسم الحاضر.
- رفع **سكرين شوت الدفع داخل البوت**.
- الأدمن يراجع الإثبات ثم يؤكد الدفع.
- بعد التأكيد يتم **إرسال الـ QR تلقائيًا** للمستخدم على تيليجرام.
- لوحة إدارة تعرض **المرجع + صورة الإثبات**.
- إزالة أي اعتماد على Paymob.

## المتطلبات
```bash
pip install -r requirements.txt
```

## الإعداد
1. انسخ `.env.example` إلى `.env`
2. املأ:
   - `TELEGRAM_BOT_TOKEN`
   - `ADMIN_CHAT_IDS`
   - `ADMIN_PASSWORD`
   - `INSTAPAY_IPA`
   - `INSTAPAY_PHONE`
   - `WALLET_PHONE`
   - `BASE_URL`

## التشغيل
```bash
python webapp.py
python bot.py
```

## رحلة المستخدم
1. المستخدم يضغط `احجز تذكرة`
2. يكتب **اسم الشخص القائم بالحجز**
3. يكتب **اسم الحاضر**
4. يكتب رقم الموبايل
5. يختار نوع التذكرة
6. يختار الدفع عبر InstaPay أو محفظة إلكترونية
7. بعد التحويل يرسل الأمر:
```bash
/ref EVT-XXXX رقم_المرجع
```
8. بعدها يرسل **سكرين شوت الدفع** داخل البوت
9. الأدمن يراجع ويؤكد الدفع
10. البوت يرسل التذكرة والـ QR

## أوامر مهمة
- عرض التذاكر:
```bash
/mytickets
```
- تحديد التذكرة التي سترفع لها الصورة:
```bash
/proof EVT-XXXX
```
- تأكيد الدفع من تيليجرام للأدمن:
```bash
/admin_paid EVT-XXXX
```

## لوحة الإدارة
- افتح:
```text
/admin?password=ADMIN_PASSWORD
```
- ستجد:
  - كل الحجوزات
  - المرجع
  - رابط صورة الإثبات
  - زر **تأكيد الدفع**
- عند الضغط على **تأكيد الدفع** يتم إرسال الـ QR تلقائيًا للمستخدم

## الفحص عند الدخول
- افتح:
```text
/scan
```
- امسح الرابط الموجود في الـ QR
- لو التذكرة مدفوعة ولم تُستخدم من قبل، يتم تسجيل الدخول مرة واحدة
